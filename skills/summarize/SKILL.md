---
name: summarize
description: |
  Use this skill to post a one-paragraph status report
  about the work just performed to a Telegram chat.
---

Stop when `WITNESS_TOKEN` or `WITNESS_CHAT` is unset or empty.
Report a single-line failure naming the missing variable.
Read the bot token from `WITNESS_TOKEN` and the chat id from `WITNESS_CHAT`.
Treat `WITNESS_CHAT` as the integer chat identifier expected by the Telegram Bot API.
Build the report only from work the earlier plugins, skills, and agents actually performed in the conversation.
Do not invent steps, files, outcomes, or links the conversation does not record.
Skip every detail about how plugins, skills, or routines were installed, loaded, authenticated, or invoked.
Exclude every scaffolding step from the report.
Focus on the key outcome, like the change delivered, the question answered, the artifact produced, the verdict, or the blocker.
Compose the message as a single paragraph naming the outcome first, the artifact second, and the next step third when one applies.
Never add headings, bullet lists, tables, quoted blocks, or horizontal rules.
Write the report as a personal status update from the agent to a human supervisor.
Open the paragraph with the salutation `Boss,`.
Phrase every clause in the first person with `I`, `me`, and `my`.
Embed the URL of the current agent run as the hyperlink target on the opening verb.
Read that URL from the environment variable the host injects, and omit the link when no such variable is set.
Open the message with exactly one emoji from the set 👍🏻, ❤️, 🍒, 😊, 🍓, ⁉️, 🧨, 👌🏻, 🌶 matching the outcome.
Use Markdown only for hyperlinks and backticked fixed-width text.
Never bold, italicize, or add headings, lists, tables, or blocks.
Set `parse_mode` to `MarkdownV2` and escape Telegram reserved characters outside inline code, link labels, and link URLs.
Keep the paragraph under one thousand characters and the whole message under four thousand.
Post the message once and feed the body via stdin so backticks and reserved characters survive the shell.
Pass `disable_web_page_preview=true` on every call.
Verify the response `ok` field is `true`.
Surface the `description` field and stop the run when `ok` is `false`.
Never name Claude, Anthropic, an AI agent, a language model, or any automation tool in the message.
Never add a `generated by` line, marketing footer, or AI-disclosure trailer.
Never log, print, echo, or write the value of `WITNESS_TOKEN` anywhere.
Stop after a single successful `sendMessage` call.
Read the four sample reports in the `examples/` directory before composing the paragraph.
